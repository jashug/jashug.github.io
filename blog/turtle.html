Turtle Language

An interpreter for a programming language combining turtle graphics and BASIC.

Context

In elementary school, every week or so we went to the computer lab. Sometimes we practiced typing or other things, but sometimes we got to play a certain game. The game had a turtle on the screen that moved in response to typed commands, and you had to move around the screen to win.

Upon learning that Python has a turtle module to do the same sort of thing, I started playing around with it at home. I could have been content to work from within Python, but I had also been thinking about programming languages more generally, including compilers and interpreters like I knew my dad worked on.

As a result, I worked on writing an interpreter in Python that would read and execute commands from a text file written in my own made-up language.

Result

In the end, I created a pretty complex language, and drew lots of pretty pictures. When I'm next in Seattle I ought to dig the source out of my backups.

My language started with simple commands to move the turtle around the screen: go forward, turn left, and so on. But without the ability to loop, drawing pictures gets boring pretty fast. I then added control constructs like loops, if-else statements, and variables, again with an arbitrary syntax inspired by BASIC (why BASIC and not Python I do not remember).

In any case, the control constructs looked nothing like the drawing constructs. The language grew organically, with me adding new ideas as I came up with them and not worrying too much about if they made sense together. Looking back, the language was a bit of a mess, but it was good enough to draw fun pictures and let me think about interesting choices programming languages make and gave me a better understanding of the difference between a compiler and an interpreter.

Resources

If you want to try this project yourself, maybe start with a program that asks the user for commands like forward 100, then runs that command using the turtle module. Once you've got that working for simple commands, have it read the commands from a file instead.

If that works, you've successfully written an interpreter. Now you can start adding features to your language to do whatever you want.
